# 岗位/职能AI转型分析平台 — 专业测试指令集

**文档版本**: v1.0  
**编制日期**: 2026-05-06  
**适用系统**: 岗位/职能AI转型智能体交互系统（ai-job-transform）  
**编制视角**: 全球顶尖产品专家 × 测试专家联合视角  

---

## 一、文档概述

本测试指令集从**产品完整性**、**技术健壮性**、**用户体验**和**商业价值**四个维度出发，为"岗位/职能AI转型分析平台"制定全面、严格、可执行的测试方案。文档覆盖功能测试、AI输出质量测试、性能测试、安全测试、可用性测试、兼容性测试、数据完整性测试和回归测试八大类别，共计 **200+ 条测试用例**，旨在确保系统在生产环境中的可靠性、安全性和用户满意度。

---

## 二、系统架构概览

| 层级 | 技术栈 | 关键组件 |
|------|--------|----------|
| 前端 | React 19 + Tailwind 4 + Framer Motion + ECharts + wouter | 13个页面路由、DashboardLayout、AppLayout、AnimatePresence |
| 通信 | tRPC 11 + SSE (Server-Sent Events) | 类型安全RPC、实时进度推送 |
| 后端 | Express 4 + Node.js 22 + multer | RESTful API + tRPC混合架构 |
| AI引擎 | 8步串行Prompt Chain + JSON Schema约束 | invokeLLM、结构化输出、工具白名单 |
| 数据库 | MySQL/TiDB + Drizzle ORM | 7张核心表（users/reports/files/invitations/ai_tools/brand_settings/report_distributions/report_feedback） |
| 存储 | S3兼容对象存储 | storagePut/storageGet |
| 认证 | Manus OAuth + JWT Session Cookie | protectedProcedure/publicProcedure |

| 空输入 | 不输入任何文本，不上传文件 | 前端阻止提交或后端返回错误 |
| 超长文本 | 输入50000字的岗位描述 | 系统正常处理或截断至合理长度 |
| 特殊字符 | 岗位名称含emoji、HTML标签、SQL关键字 | 正确存储和显示，无注入 |
| 纯英文输入 | 全英文JD | 系统正常分析，报告可中英混合 |
| 多语言混合 | 中英日韩混合文本 | 不崩溃，核心分析基于中文 |
| 重复提交 | 快速连续点击"开始分析"3次 | 只创建1份报告，不产生重复 |

### 12.2 网络异常

| 场景 | 模拟方法 | 预期行为 |
|------|----------|----------|
| SSE连接中断 | 分析过程中断网 | 前端检测到连接关闭，启用模拟进度或提示重连 |
| 上传中断 | 大文件上传中断网 | 明确的错误提示，不产生残留数据 |
| API超时 | 服务器响应>30s | 前端显示超时提示，不无限等待 |
| 服务器500 | 后端抛出未捕获异常 | 前端显示友好错误信息，不暴露堆栈 |

### 12.3 状态异常

| 场景 | 触发方式 | 预期行为 |
|------|----------|----------|
| 访问不存在的报告 | /report/invalid-id | 显示404或友好提示 |
| 访问analyzing状态的报告 | 分析未完成时直接访问报告URL | 重定向至进度页或显示"分析中"状态 |
| 重复接受邀请 | 同一用户多次点击邀请链接 | 第二次提示"邀请码已使用" |
| 删除后访问 | 删除报告后通过浏览器后退访问 | 显示"报告不存在"而非崩溃 |
