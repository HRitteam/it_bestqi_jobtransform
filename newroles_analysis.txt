Analysis: Why "转型后新增" is always empty

ROOT CAUSE: The step7 schema does NOT include a "newRoles" field!

The schema at line 573 requires: ["taskClassification", "newCapabilityModel", "roadmap", "toolRecommendations"]
There is NO "newRoles" field in the schema definition.

The OrgChangeChart component expects:
- taskClassification (exists in data) ✓
- newRoles (NEVER generated by LLM because not in schema) ✗

The prompt says "任务重新分类" but never asks for "新增角色/新增岗位".

FIX OPTIONS:
1. Add "newRoles" to the step7 schema and prompt (requires re-generating reports)
2. Derive "new roles" from the existing data (newCapabilityModel or taskClassification)

Option 2 is better because it works with existing reports without re-generation.

The "newCapabilityModel" field contains new skills/capabilities needed after transformation.
We can use these to synthesize "new roles" in the frontend display.

Better approach: Instead of showing "new roles" (which don't exist), show the 
"newCapabilityModel" items as "转型后新增能力/角色" since they represent what's NEW 
after transformation. This makes the right side useful and informative.

Also add newRoles to the schema for future reports.
